이 블로그는 모든 글을 마크다운 파일로 작성해서 올리고 있다. 이 게시글은 내가 마크다운을 작성하면서 유용하게 쓴 문법을 하나씩 넣어가며 계속 업데이트할 생각이다.


## 1. 마크다운을 사용한 제목 붙이기
```md
# h1        글의 제목
## h2       글의 부제목 혹은 소제목
### h3      
#### h4     
##### h5    
###### h6   아래로 내려올수록 크기가 작아진다.
```

마크다운을 사용하면 `#`의 개수를 더하는 것만으로 쉽게 글의 위계를 작성할 수 있다. `#`이 하나 있는 것이 가장 큰 제목이 되고 `#`을 더할수록 작아진다. 크기도 그렇지만 글의 위계도 `#`으로 정해지는데, SEO는 이를 바탕으로 글의 위계를 파악하게 된다. 예를 들어 구글 검색결과에는 글자 크기만 키운 '제목' 대신 `# 제목`이 출력된다. 


## 2. 마크다운에서 줄바꿈 (엔터)

```md
이 줄을 쓴 다음에 줄바꿈을 하고 싶다고 해봅시다.
이렇게 쓰면 줄바꿈이 적용이 안됩니다.

마크다운에서는 단락 끝에서 두번 띄어쓰기를 해야됩니다.VV
이렇게 써야해요.
```

> 이 줄을 쓴 다음에 줄바꿈을 하고 싶다고 해봅시다.
> 이렇게 쓰면 줄바꿈이 적용이 안됩니다.
> 
> 마크다운에서는 단락 끝에서 두번 띄어쓰기를 해야됩니다.  
> 이렇게 써야해요.

마크다운은 줄바꿈이 상당히 비직관적인데 이전 단락을 그냥 끝내면 안되고 끝에서 두번 띄어줘야 띄어쓰기가 된다. VS Code나 Typora 같은 몇몇 에디터는 엔터를 하면 자동 줄바꿈을 지원하기도 한다. 해당 방식이 익숙하지 않다면 에디터를 바꿔보자. 


## 3. 마크다운에서 여러 줄 바꿈 (여러 줄 엔터)

```md
글을 쓰고 한 두어줄 띄고 다음 글을 쓰고 싶을 때가 있잖아요?



그렇다고 여러번 엔터를 쳐도 그게 적용되진 않습니다.  
<br>
<br>
<br>
이렇게 사이에 HTML 코드를 넣어주는 식으로 임시로 해결하고 있습니다.
```

>글을 쓰고 한 두어줄 띄고 다음 글을 쓰고 싶을 때가 있잖아요?
>
>
>
>그렇다고 여러번 엔터를 쳐도 그게 적용되진 않습니다.
> <br>
> <br>
> <br>
>이렇게 사이에 HTML 코드를 넣어주는 식으로 임시로 해결하고 있습니다.

개인적으론 이게 제일 불편하다. 여러 번 엔터치면 그냥 그만큼 줄을 띄어줬으면 좋겠는데. 위의 마크다운 코드를 보면 알겠지만 엔터를 두세번 쳐도 실제 문서에서는 한 번만 줄바꿈한 것으로 인식한다. 애초에 마크다운에서 지원을 안한다. 정 원한다면 `<br>` HTML 태그를 써서 강제로 줄바꿈해줄 수는 있다.


## 4. 문법 강조 Syntax Highlighting
    ```python
    import numpy as np
    import pandas as pd

    a = np.array([[1,2],[3,4]])
    print(np.corrcoef(a))
    ```
> ```python
> import numpy as np
> import pandas as pd
> 
> a = np.array([[1,2],[3,4]])
> print(np.corrcoef(a))
> ```

개발자들이 깃허브로 블로그를 만드는 이유 중 하나가 프로그래밍 코드에 대한 문법 강조가 간단하게 된다는 장점 때문이다. 위의 코드처럼 사용하는 언어를 명시해주면 에디터에서 보는 것처럼 문법 강조 기능을 마크다운을 통해서도 간단히 구현할 수 있다.

한가지 주의할 점으로 제일 처음과 마지막의 점 세개는 작은 따옴표 `'`가 아니다. 키보드 상에서는 숫자키 1 옆에 ~ 표시랑 같이 있는 키다. 구글에 쳐보니 억음 부호*grave accent*라고 하는데 프로그래머들은 백틱*backtick*이라고 부르는 듯 하다.


## 5. KaTeX 문법
```markdown
$$
E = mc^2
$$
```
> $$
> E = mc^2
> $$

글을 쓰다보면 수식을 넣을 필요가 있을 때가 있다. 여태까지는 수식을 워드 등의 다른 프로그램에서 작성한 뒤 이미지 파일 형태로 붙여넣었지만 TeX 문법을 사용하면 그럴 필요가 없다. 

TeX 자체는 마크다운이 기본적으로 지원하는 기능이 아니지만 많은 플러그인이 TeX 렌더링을 지원한다. TeX 문법도 LaTeX, KaTeX 등 나는 KaTeX를 사용하고 있다.

TeX 문법 자체를 새로 배워야하는 점이 단점이라면 단점일까? 그래도 주로 쓰는 수식은 정해져 있기에 금방 익숙해진다. TeX 문법은 마크다운처럼 [새로운 글]()을 써서 틈틈이 정리하기로 했다.


## 6. 링크 삽입
```markdown
[내 GitHub 프로필](https://github.com/akaiuun12)
```
> [내 GitHub 프로필](https://github.com/akaiuun12)

마크다운을 사용하면 링크를 쉽게 삽입할 수 있다. 대괄호 `[]` 사이에 설명을 넣고 괄호 `()`안에 주소를 넣어주면 쉽게 링크를 생성할 수 있다. Github의 rawfile 주소 등을 넣으면 파일 다운로드 링크가 된다.


## 7. 이미지 삽입
```markdown
![이미지 이름](이미지경로)
```
> ![이미지 이름](https://res.cloudinary.com/dhchweuhy/image/upload/c_fill,w_380,h_200,r_5,f_auto,q_auto/lrc/20200623_img1)

마크다운에 이미지를 넣는 방법은 링크를 넣는 방법과 유사하다. 다른 점은 앞에 `!`를 붙여준다는 것이다. 이 방법은 아주 단순한 반면 세부적 설정을 할 수 없다. 예를 들어 여러 이미지를 한 행에 나열하거나 크기 조정을 할 수 없다.


## 8. 인용 표시
```markdown
> 다른 사람의 말을 가져온다면, 자신이 한 말이 아닙니다.  
> 그것이 인용이니까.
```
> 다른 사람의 말을 빌려온다면, 자신이 한 말이 아닙니다.  
> 그것이 인용이니까.

인용은 `>`로 쓴다. 물론 이 글에서 쓰는 것처럼 꼭 인용의 의미로만 써야하는 것은 아니다. 대부분의 테마는 인용(blockquote)를 시각적으로 구분해주기 떄문에 강조나 분리하고 싶을 때 사용할 수도 있다. 


## 9. 마크다운에서 표 만들기
```markdown
|이름|소속|설명|
|---|---|---|
|해리|그리핀도르|안경이 잘 어울림|
|말포이|슬리데린|자주 입을 다뭄|
```

> |이름|소속|설명|
> |---|---|---|
> |해리|그리핀도르|안경이 잘 어울림|
> |말포이|슬리데린|자주 입을 다뭄|

마크다운에서는 `|` 표시를 사용해 간단하게 표를 만들 수 있다. `|`는 표의 각 셀을 구분하는 역할을 한다. 줄을 바꾸면 알아서 행을 바꿔준다.


## 10. 글꼴 설정 (이탤릭, 볼드)
```
글을 쓸 때 *italic*은 설명을
**bold**는 강조를 의미한다.
***bold***
```

> 글을 쓸 때 *italic*은 설명을  
> **bold**는 강조를 의미한다.

이탤릭은 `*` 하나로, 볼드는 `*` 두 개로 감싸주면 된다.
