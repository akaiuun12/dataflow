Jupyter는 파이썬 코드와 결과, 줄글까지 한 파일에 저장할 수 있다. 파일 트리를 만들고 정리하는 것도 물론 중요하지만, 실험과정 및 분석 방법을 설명하고 정리하는 데이터 과학에서는 Jupyter를 특히 많이 사용한다. 

## Jupyter 설치
```sh
conda install jupyterlab
```

## `# %%`로 py파일에서도 코드 블럭 사용하기
```python
# %% 
import numpy as np
```

%로 주피터 노트북에 주는 명령을 매직 커맨드*Magic Command*라고 한다. 매직 커맨드 `# %%`를 사용하면 일반 `.py` 파일에서도 코드 블럭 별 실행이 가능해진다. VS Code의 왼쪽에서는 코드를 짜고 오른쪽에서 결과를 확인할 수 있게 된다.


## Enter: 명령 모드 (Command Mode)

| 명령 | 설명 | Windows | Mac| 
| --- | --- | --- | --- |
| Run Cells | 해당 셀 실행 | Ctrl + Enter | Cmd + Return |
| Split Cell | 셀을 두개로 분할 | Ctrl + Shift + '-' | Cmd + K, Cmd + Shift + \ |


## Esc: 편집 모드 (Edit Mode)

| 명령 | 설명 | Windows | Mac| 
| --- | --- | --- | --- |
| Cell Type - Markdown  | 마크다운 셀로 변환    | M | M |
| Cell Type - Code      | 코드 셀로 변환        | Y | Y |
| Delete Cells          | 셀 삭제              | DD | DD |
| Insert Cell Below     | 아래에 새로운 셀 삽입 | B | B |


## 가상환경 커널 지우기
```bash
{user}/Library/Jupyter/kernel/ 
```
conda로 가상환경을 만들면 각 가상환경 마다 서로 다른 버전의 파이썬을 쓴다. 이걸 커널이라고 하는데 보통은 가상환경을 지우면 같이 사라진다. 근데 가상환경 명을 동일하게 했더니 가상환경은 지워져도 커널은 남아 제대로 실행이 안되더라. 가상환경은 미니콘다 쪽에 설치/제거되고 커널은 Jupyter쪽에 저장되는 것이 이유였다. 위의 위치에 있는 커널을 보고 지워주면 말끔히 사라진다.
